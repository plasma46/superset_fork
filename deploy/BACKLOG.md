## ✅ BACK-2 — Баг: ввод в одну ячейку проставляется во все строки

**Цель:** При редактировании поля комментария в строке N значение не должно появляться в остальных строках.

**Контекст:**
Судя по скриншоту: пользователь вводит "Fhhfg" в строку 1, и это значение визуально отображается во всех строках колонки `comment_text`. Это классический баг shared state — dirty-state хранится по `view_column` без учёта ключа строки, либо ag-grid cellRenderer не изолирован по rowIndex.

**Где смотреть:**
- `src/AgGridTableChart.tsx` — как формируется `dirtyState` и как значение передаётся в `cellRenderer`/`cellEditor`.
- Проверить: `dirtyState` — это `Record<string, Record<string, unknown>>` где ключ — composite row key. Если ключ формируется неправильно (например, всегда пустая строка), все строки получат одно значение.
- `src/utils/commentEditing.ts` — функция `buildRowKey` / `makeCompositeKey`.

**Гипотезы:**
1. `key_mapping` не настроен → `buildRowKey` возвращает `""` для всех строк → все строки обновляются одним ключом.
2. `cellValueGetter` или `valueGetter` в колонке-комментарии читает из `dirtyState` по `view_column` без строчного ключа.
3. ag-grid `cellEditor` shared instance (не per-row).

**Критерий готовности:** Ввод в строку N не затрагивает строки M. Unit-тест на `buildRowKey` с двумя разными строками.
